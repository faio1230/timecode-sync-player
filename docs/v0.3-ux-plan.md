# v0.3 UX パック: 状態可読性の改善

作成: 2026-07-18（仕様確定済み）。背景: ギャップ演出+信号断+表示保持が重なると
オペレーターが状態を読めない（2026-07-17 の実体験。progress の QA-002 ソーク節参照）。
**実行ルールは docs/test-coverage-roadmap.md の「一括実行ルール」に従う。** 上書き事項:
- ブランチ: feature/v0.3-ux。push しない。E2E ゲートは全件（実機込み・Skip なし）
- 表示のみの変更であること: 同期・pause 所有権・ギャップ判定等の**挙動ロジックには一切触れない**
  （既存の所有権/遷移テストが無変更で通ることが必須）

## X1: 信号断中のタイムコード表示（淡色化 + NO SIGNAL）

- LTC 監視中に信号断（LtcSignalLossPolicy が Lost 判定。モードがランスルーでも表示は変える）に
  なったら: タイムコード文字色をグレー（例 #666、既存の淡色系と統一）へ、`LtcFormatText` を
  「NO SIGNAL」表示へ切替。最終タイムコード値の保持は仕様どおり維持
- 信号復帰（既存のヒステリシス判定）で通常表示に戻す
- 手動 STOP は従来どおり（「LTC 停止中」系の既存表示。信号断表示とは区別）
- 表示判定は純クラス（例 LtcDisplayStateFormatter）に切り出しユニットテスト

## X2: ギャップ中のトラック表示（次トラック情報）

- `CurrentTrackLabel` のギャップ表示を「Gap: Black → track2 @ 00:13:00」形式に拡張
  （次トラック名 + タイムライン開始時刻。時刻は TC 形式 HH:MM:SS で十分、フレームは不要）
- 後続トラックが無い場合は「Gap: Black (以降なし)」
- NoTracks 時の表示も同形式で整合させる
- フォーマッタは既存 `PlaylistCurrentTrackLabelFormatter` の拡張として実装しユニットテスト

## X3: 信号断 pause の理由表示

- 信号断（Stop モード）による自動 pause 中は、再生ボタン近傍に小さく「信号断で停止中」を表示
  （新規 TextBlock、通常時は非表示/空。AutomationId 付与）
- 表示条件は LtcSignalLossPolicy の pause 所有権状態から導出（**所有権ロジック自体は変更しない**。
  読み取り用の公開が必要なら read-only プロパティ追加のみ可）
- 手動 pause・ギャップ pause では表示しない（所有者の区別がそのまま表示に出る）
- 信号復帰の自動再開・手動 Play で消える

## テスト要件

- ユニット: 各フォーマッタ/表示判定の全分岐（信号断/復帰/手動STOP/ギャップ有無/後続なし/
  pause 所有者別）+ ミューテーション確認
- 結合（SyncScenarioHarness）: 表示状態を Effects 経由で記録し、信号断→復帰・ギャップ進入→脱出の
  シナリオで表示遷移が仕様どおりであること（既存シナリオへの追加アサートで可）
