# 機能プラン: ミュートボタン + 音量スライダー

作成: 2026-07-17（仕様確定済み）。v0.2.0 に同梱する最後の機能。
**実行ルールは docs/test-coverage-roadmap.md の「一括実行ルール」に従う。** 上書き事項:
- ブランチ: feature/volume-control。push しない
- E2E ゲートは全件（実機 LTC 11件含む・Skip なし）

## 背景

無音で再生したい現場が多い（映像はプロジェクター、音は別系統等）。mpv の `mute` / `volume`
プロパティを UI から制御できるようにする。

## 仕様

| 項目 | 内容 |
|---|---|
| UI | 再生コントロール付近に**ミュートトグルボタン**（状態がラベルで分かる。Spout/Timeline ボタンと同じトグル流儀、AutomationId 付与）と**音量スライダー**（0〜100、AutomationId 付与） |
| mpv 制御 | `mute`（yes/no）と `volume`（0〜100）プロパティ。既存の SetPropertyString パターンを踏襲し新しい呼び出し方式を作らない |
| 永続化 | 両方 AppSettings（`IsMuted` / `Volume`、既定 false / 100、volume は 0〜100 クランプ）。起動時に mpv セッション初期化後へ適用 |
| 挙動 | ミュート中もスライダー操作可（値は保持、解除時に反映）。ミュート/音量は**再生状態と独立**（pause/シーク/トラック切替/ギャップで変化しない） |

## テスト要件（全層必須 — 遷移横断が主眼）

1. **ユニット**: 状態・クランプ・永続化・プロパティ書き込み判定（ミュート時のトグルラベル等）
2. **結合（SyncScenarioHarness 拡張）**: フェイク mpv のプロパティ書き込み記録を使い、
   Single / Continue 両モードで以下の遷移を跨いで
   (a) mute/volume の論理状態が変化しないこと
   (b) 遷移処理中に `mute` / `volume` プロパティへの**意図しない書き込みが発生しない**こと
   を証明する:
   - トラック切替（Continue 自動切替・手動 次/前）
   - ギャップ進入/脱出（Freeze・Black 両方、GapBehavior 切替またぎも1ケース）
   - 信号断 pause / 自動復帰（Stop モード）
   - プロジェクト読込・StopPlayback・LoadFile・シーク
3. **E2E（実アプリ）**: トグル/スライダー操作が mpv に反映（音声はアサートできないため
   ボタンラベル・スライダー値・設定ファイルで検証）+ アプリ再起動後に復元
4. **実機 E2E（VB-CABLE）**: Continue + 実 LTC 追従中に「ミュート ON → ギャップ跨ぎ →
   トラック切替」の1シナリオでミュート状態（ラベル + 設定値）が維持される
5. ミューテーション確認（各層最低1件、計3件以上）

## 制約

- LoadFile 経路が per-file オプションで音声状態を上書きしないこと（mpv の mute/volume は
  プレイヤープロパティなので通常維持されるが、結合テストで証明する）
- 既存ログ・既存挙動不変（音量系の追加ログは可）
- docs/settings.md へ新キー2つを追記。CHANGELOG 0.2.0 の Added に追記
- 実バグを発見した場合は従来どおり停止して報告
