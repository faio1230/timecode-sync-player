## Q1: プロジェクト読込は停止状態で復元（QA-001・仕様決定済み）

- プロジェクト読込（UI ボタン・`--open`/`--load-project` CLI とも）完了後、先頭トラックを
  **ロードして先頭フレームを表示したまま pause** とする。再生開始は明示操作（Play）または
  Sync ON 時の同期エンジンによるもののみ
- 対話的な「開く」「プレイリストへ追加」での**新規動画の追加**は現状（即再生）を維持する —
  変更するのは「保存済みプロジェクトの復元」経路のみ
- テスト: 結合（ハーネス: 読込後に pause=yes のまま・play 書き込みが読込経路で発生しない）+
  E2E（SystemScenario のラウンドトリップに「読込後は停止している」アサートを追加）

## Q2: 設定永続化の配線完成（QA-007/003/004/005・同根）

現状 `syncMode` / `gapBehavior` / `ltcDeviceIndex` / `lastOpenedProjectPath` は settings.json と
docs/settings.md に存在するが、保存も復元もされていない（死にキー）。以下を配線する:

1. **syncMode / gapBehavior**: UI 変更時に UpdateAsync で保存（Mute 等と同じ流儀）、起動時に
   UI へ復元。docs/settings.md の既定値表記と実挙動を一致させる（AppSettings.Default と
   ViewModel 初期値の不一致も解消 — 既定は現行 UI の実際の初期値に合わせ、docs を修正する
   方向でよい。数値enumとラベルの対応表も docs に追記）
2. **LTC デバイス**: インデックスは列挙順依存で壊れやすいため、**キーを `ltcDeviceName`
   （文字列・既定空）に置き換える**。保存はユーザーがコンボで選択した時、復元は起動時の
   デバイス一覧から名前一致（見つからなければ従来どおり先頭 + 未選択扱いにはしない現行動作を
   維持しログに記録）。旧キー `ltcDeviceIndex` は読み捨てて削除（後方互換: 存在しても無視）
3. **lastOpenedProjectPath**: プロジェクトの読込成功・保存成功時に保存する（復元動作 —
   起動時の自動読込 — は**追加しない**。キーの記録のみ。用途は将来の「最近使ったプロジェクト」）
4. docs/settings.md を全キー実挙動と一致するよう更新

- テスト: ユニット（保存/復元/旧キー無視/名前不一致フォールバック）+ 結合（設定復元が
  各 Coordinator の初期状態に反映）+ E2E（変更→正常終了→再起動→復元、QA-007 の再現手順そのもの）
